## 7. Key Design Decisions

### 7.1 Why Tiptap for Editor?
- Berbasis ProseMirror (production-proven)
- Custom extension support (untuk highlight-to-followup)
- Headless вҖ” full control atas UI
- Active community & documentation

### 7.2 Why Sanctum (bukan Passport)?
- Lebih ringan untuk SPA authentication
- Cookie-based session вҖ” cocok untuk same-domain setup
- Tidak perlu OAuth complexity

### 7.3 Why Next.js (bukan React SPA biasa)?
- SSR untuk SEO & initial load performance
- App Router untuk layout nesting
- Built-in API routes jika diperlukan (BFF pattern)
- Excellent PWA support

### 7.4 Why Zustand (bukan Redux)?
- Minimal boilerplate
- No provider wrapping needed
- Perfect for medium-complexity state
- TypeScript-first

### 7.5 Polling vs WebSocket untuk Real-Time
- **Phase 1**: Polling via TanStack Query (refetchInterval: 5000ms)
- **Phase 2 (optional)**: Laravel Reverb/WebSocket untuk true real-time
- Polling lebih simple untuk MVP, cukup performant untuk 500 users
